# Contributing

Thanks for improving Hook Skills.

## What To Contribute

Good contributions usually improve one of these:

- clearer hook mechanism boundaries
- better product reveal timing rules
- sharper visual opening guidance
- stronger benchmark distillation
- cleaner handoffs between skills
- tighter QA checklists

## What Not To Contribute

Please do not turn this repo into:

- a generic copywriting library
- a broad ad-creative swipe file
- a niche-specific script database
- a collection of empty trend language

## Contribution Standard

A strong contribution should make the repo more operational, not just longer.

Before proposing a change, ask:

1. Does this make the skill more reusable?
2. Does this reduce downstream guessing?
3. Does this improve opening strength, timing clarity, or failure diagnosis?
4. Is this still about hooks?

## Preferred Change Types

- improve a mechanism definition
- tighten a failure mode
- add a more diagnostic example
- clarify when to hand off to another skill
- improve method write-ups with better evidence framing

## File Design

Keep the main `SKILL.md` concise.

If extra detail is needed, prefer:

- `examples.md`
- `hook-principles.md`
- a small reference file in the same skill directory

## Voice

Write in plain English.

Avoid:

- inflated marketing language
- fake certainty
- generic "viral" claims
- empty aesthetic adjectives
